APM version 8.7.0edit

Breaking Changesedit
  • transaction.failure_count has been removed. transaction.success_count type has changed to aggregated_metric_double 9791
  • transaction.success_count has been moved to event.success_count 9819
  • Stop indexing transaction metrics to metrics-apm.internal 9846
  • Stop indexing span destination metrics to metrics-apm.internal 9926
  • apmserver.aggregation.txmetrics.overflowed metric has been renamed to apmserver.aggregation.txmetrics.overflowed.total 10330
Bug fixesedit
  • We no longer unconditionally block shutdown for 5 seconds when tail-based sampling is enabled 9838
  • Fix a panic caused by accepting new grpc connections during shutdown 10013
Intake API Changesedit
  • Add request.id to span.context.http 9673
  • Add agent.activation_method metadata field 9850
Addededit
  • Add event.success_count to transaction events and transaction metrics 9819 10165
  • Set _doc_count for service destination metrics 9931
  • Add transaction.duration.summary to transaction metrics 9973 10214
  • Add service.language.name to service transaction metrics and service summary metrics 10111
  • Add global labels to service destination metrics 10056
  • Improve APM UI query performance by producing 1m, 10m and 60m aggregation intervals for transaction metrics. Store them into dedicated data streams metrics-apm.transaction.${interval} 9846
  • Improve APM UI query performance by producing 1m, 10m and 60m aggregation intervals for service destination metrics. Store them into dedicated data streams metrics-apm.service_destination.${interval} 9926
  • Improve APM UI query performance by producing 1m, 10m and 60m aggregation intervals for service metrics. Store them into dedicated data streams metrics-apm.service.${interval} 9960
  • Add histogram support to service metrics aggregations 9864 10214
  • Add a service summary metrics aggregator that writes to metrics-apm.service_summary.${interval} with 1m, 10m and 60m intervals 10029
  • Service metrics and service summary aggregations are now always enabled 10060 10109
  • Dedicated overflow buckets for transaction, service, service destination and service summary aggregation to limit cardinality 10069 9856 10061 10362
  • Automatically scale MaxGroups, MaxTransactionGroups and MaxServices based on available memory 9856 10157
  • Set data streams with non-default rollup intervals as hidden 10070
  • Rename service metrics to service transaction metrics 10095
  • Reduce memory usage of transaction and service transaction aggregations 10178
  • Metrics data streams now use synthetic source 9756
  • Translate otel log events to error events 10066
  • Use Elasticsearch fetcher for agent remote configuration where possible 9720 10088
  • Fetch sourcemaps from Elasticsearch 9722
  • Add overflow metrics for transaction, span metrics, service transaction and service summary metrics 10330
