Add connectors

edit

This functionality is in technical preview and may be changed or removed in a future release. Elastic will work to fix any issues, but features in technical preview are not subject to the support SLA of official GA features.

You can add connectors to cases to push information to these external incident management systems:

  • IBM Resilient
  • Jira
  • ServiceNow ITSM
  • ServiceNow SecOps
  • Swimlane
  • Webhook - Case Management

To create connectors and send cases to external systems, you must have the appropriate Kibana feature privileges. Refer to Configure access to cases.

Create connectors

edit

You can create connectors in Management > Stack Management > Rules and Connectors, as described in Connectors. Alternatively, you can create them in Management > Stack Management > Cases:

  1. Click Edit external connection.

  2. From the Incident management system list, select Add new connector.
  3. Select an external incident management system.
  4. Enter your required settings. Refer to IBM Resilient, Jira, ServiceNow ITSM, ServiceNow SecOps, Swimlane, or Webhook - Case Management for connector configuration details.
  5. Click Save.

Edit connector settings

edit

You can create additional connectors, update existing connectors, change the default connector, and change case closure options.

  1. Go to Management > Stack Management > Cases, click Edit external connection.
  2. To change whether cases are automatically closed after they are sent to an external system, update the case closure options.
  3. To change the default connector for new cases, select the connector from the Incident management system list.
  4. To update a connector, click Update <connector name> and edit the connector fields as required.
