Tune Elasticsearch

edit

This documentation refers to the standalone (legacy) method of running APM Server. This method of running APM Server will be deprecated and removed in a future release. Please consider upgrading to the Elastic APM integration. If you’ve already upgraded, please see Tune Elasticsearch instead.

The Elasticsearch reference provides insight on tuning Elasticsearch.

Tune for indexing speed provides information on:

  • Refresh interval
  • Disabling swapping
  • Optimizing filesystem cache
  • Considerations regarding faster hardware
  • Setting the indexing buffer size

Tune for disk usage provides information on:

  • Disabling unneeded features
  • Shard size
  • Shrink index
