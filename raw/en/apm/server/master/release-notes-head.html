APM Server version HEADedit

View commits

Breaking Changesedit

  • network.connection_type is now network.connection.type 5671
  • transaction.page and error.page no longer recorded 5872
  • [experimental] This breaking change applies to the experimental tail-based sampling feature. apm-server.sampling.tail now requires apm-server.data_streams.enabled 5952
  • [beta] This breaking change applies to the beta APM integration (Elastic Agent). The traces-sampled-* data stream is now traces-apm.sampled-* 5952

Bug fixesedit

  • Fix apm_error_grouping_name and apm_convert_destination_address 5876
  • corrected OTel attribute names for net.host.connection.* 5671
  • Fix response to agent config when running under Fleet with no agent config defined 5917

Intake API Changesedit

  • network.connection.type was added to stream metadata 5671

Addededit

  • service_destination span metrics now take into account composite spans 5896
  • add zero-downtime config reloads via SO_REUSEPORT 5911
  • experimental support for writing data streams in standalone mode 5928
  • Data streams now define a default dynamic mapping parameter, overridable in the <data-stream>@custom template 5947
  • The error.log.message or error.exception.message field of errors will be copied to the ECS field message 5974
