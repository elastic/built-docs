Configure Functionbeat to use security featuresedit

If you want Functionbeat to connect to a cluster that has security features enabled, there are extra configuration steps:

  1. Grant users access to secured resources.

    You can use role-based access control to grant Functionbeat users access to secured resources.

  2. Grant access using API keys.

    You can use API keys to grant access to Elasticsearch more securely.

  3. Configure authentication credentials.

    To interact with a secured cluster, Functionbeat must either provide basic authentication credentials or present a client certificate.

  4. Configure Functionbeat to use encrypted connections.

    If encryption is enabled on the cluster, you need to enable HTTPS in the Functionbeat configuration.

For more information about security features, see Secure a cluster.
