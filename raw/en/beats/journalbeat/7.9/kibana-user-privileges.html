Grant privileges and roles needed to read Journalbeat data from Kibanaedit

Kibana users typically need to view dashboards and visualizations that contain Journalbeat data. These users might also need to create and edit dashboards and visualizations.

To grant users the required privileges:

  1. Create a reader role, called something like journalbeat_reader, that has the following privilege:

    Type Privilege Purpose

    Index

    read on journalbeat-* indices

    Read data indexed by Journalbeat

    Spaces

    Read or All on Dashboards, Visualize, and Discover

    Allow the user to view, edit, and create dashboards, as well as browse data.

    Spaces

    Read or All on Kibana Logs

    Allow the use of Kibana Logs

  2. Assign the reader role, along with the following built-in roles, to users who need to read Journalbeat data:

    Role Purpose

    monitoring_user

    Allow users to monitor the health of Journalbeat itself. Only assign this role to users who manage Journalbeat.
