Start Packetbeat

edit

Start Packetbeatedit

Before starting Packetbeat:

  • Follow the steps in Quick start: installation and configuration to install, configure, and set up the Packetbeat environment.
  • Make sure Kibana and Elasticsearch are running.
  • Make sure the user specified in packetbeat.yml is authorized to publish events.

To start Packetbeat, run:

sudo service packetbeat start

If you use an init.d script to start Packetbeat, you can’t specify command line flags (see Command reference). To specify flags, start Packetbeat in the foreground.

Also see Packetbeat and systemd.
