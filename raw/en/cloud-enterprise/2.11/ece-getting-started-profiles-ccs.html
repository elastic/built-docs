Cross-cluster search profileedit

This profile manages remote connections for running Elasticsearch queries across multiple deployments and indices. These federated searches make it possible to break up large deployments into smaller, more resilient Elasticsearch clusters. You can organize deployments by departments or projects for example, but still have the ability to aggregate query results and get visibility into your Elastic Cloud Enterprise infrastructure. You can add remote connections either when you create your deployment or when you customize it. To know more about cross-cluster search, see Enable cross-cluster search.

Included in this profile:

  • Elasticsearch cross-cluster search nodes: Start at 1 GB memory x 1 availability zone. Uses the ccs.default instance configuration.
  • Kibana: Starts at 1 GB memory x 1 availability zone. Uses the kibana instance configuration.
