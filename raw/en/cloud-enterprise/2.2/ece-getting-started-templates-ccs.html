Cross-cluster search templateedit

This template manages remote connections for running Elasticsearch queries across multiple deployments and indices. These federated searches make it possible to break up large deployments into smaller, more resilient Elasticsearch clusters. You can organize deployments by departments or projects for example, but still have the ability to aggregate query results and get visibility into your Elastic Cloud Enterprise infrastructure.

Included in this template:

  • Elasticsearch:

    • Cross cluster search nodes: Start at 1 GB memory x 1 availability zone. Uses the ccs.default instance configuration.
  • Kibana: Starts at 1 GB memory x 1 availability zone. Uses the kibana instance configuration.
