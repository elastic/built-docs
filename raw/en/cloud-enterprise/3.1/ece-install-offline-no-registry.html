Without a private Docker registryedit

To perform an offline installation without a private Docker registry, you have to download the available Docker Images on each host.

  1. On an internet-connected host that has Docker installed, download the Available Docker Images. Note that for ECE version 3.0, if you want to use Elastic Stack version 8.0 in your deployments, you need to download and make available both the version 7.x and version 8.x Docker images (the version 7.x images are required for system clusters).

    docker pull docker.elastic.co/cloud-enterprise/elastic-cloud-enterprise:3.1.0
docker pull docker.elastic.co/cloud-assets/elasticsearch:7.17.1-0
docker pull docker.elastic.co/cloud-assets/kibana:7.17.1-0
docker pull docker.elastic.co/cloud-assets/apm:7.17.1-0
docker pull docker.elastic.co/cloud-assets/enterprise-search:7.17.1-0
docker pull docker.elastic.co/cloud-release/elasticsearch-cloud-ess:8.1.0
docker pull docker.elastic.co/cloud-release/kibana-cloud:8.1.0
docker pull docker.elastic.co/cloud-release/elastic-agent-cloud:8.1.0
docker pull docker.elastic.co/cloud-release/enterprise-search-cloud:8.1.0

    For example, for Elastic Cloud Enterprise 3.1.0 and the Elastic Stack versions it shipped with, you need:

    • Elastic Cloud Enterprise 3.1.0
    • Elasticsearch 8.1.0, Kibana 8.1.0, APM 8.1.0, and Enterprise Search 8.1.0

  2. Create .tar files of the images:

    docker save -o ece.3.1.0.tar docker.elastic.co/cloud-enterprise/elastic-cloud-enterprise:3.1.0
docker save -o es.7.17.1-0.tar docker.elastic.co/cloud-assets/elasticsearch:7.17.1-0
docker save -o kibana.7.17.1-0.tar docker.elastic.co/cloud-assets/kibana:7.17.1-0
docker save -o apm.7.17.1-0.tar docker.elastic.co/cloud-assets/apm:7.17.1-0
docker save -o enterprise-search.7.17.1-0.tar docker.elastic.co/cloud-assets/enterprise-search:7.17.1-0
docker save -o es.8.1.0.tar docker.elastic.co/cloud-release/elasticsearch-cloud-ess:8.1.0
docker save -o kibana.8.1.0.tar docker.elastic.co/cloud-release/kibana-cloud:8.1.0
docker save -o apm.8.1.0.tar docker.elastic.co/cloud-release/elastic-agent-cloud:8.1.0
docker save -o enterprise-search.8.1.0.tar docker.elastic.co/cloud-release/enterprise-search-cloud:8.1.0
  3. Copy the .tar files to a location on your network where they are available to each host where you plan to install Elastic Cloud Enterprise. Alternatively, you can copy the .tar files to each host directly.

  4. On each host, load the images into Docker, replacing FILE_PATH with the correct path to the .tar files:

    docker load < FILE_PATH/ece.3.1.0.tar
docker load < FILE_PATH/es.7.17.1-0.tar
docker load < FILE_PATH/kibana.7.17.1-0.tar
docker load < FILE_PATH/apm.7.17.1-0.tar
docker load < FILE_PATH/enterprise-search.7.17.1-0.tar
docker load < FILE_PATH/es.8.1.0.tar
docker load < FILE_PATH/kibana.8.1.0.tar
docker load < FILE_PATH/apm.8.1.0.tar
docker load < FILE_PATH/enterprise-search.8.1.0.tar
  5. Optional: Remove the .tar files after installation.

  6. On an internet-connected host, download the installation script:

    curl -L -O https://download.elastic.co/cloud/elastic-cloud-enterprise.sh
  7. Copy the installation script to each host where you plan to install Elastic Cloud Enterprise or make it available on your network.

  8. Invoke the installation script on each host:

    1. On the first host:

      bash elastic-cloud-enterprise.sh install

    2. On additional hosts, include the --coordinator-host HOST_IP and --roles-token 'TOKEN' parameters provided to you when you installed on the first host:

      bash elastic-cloud-enterprise.sh install
  --coordinator-host HOST_IP
  --roles-token 'TOKEN'
