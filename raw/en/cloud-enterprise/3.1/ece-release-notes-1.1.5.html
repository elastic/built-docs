Elastic Cloud Enterprise 1.1.5edit

New for Elastic Cloud Enterprise version 1.1.5:

  • Elasticsearch and Kibana 6.4.0 and 5.6.11. This version of Elastic Cloud Enterprise includes our latest Elastic Stack versions.
  • Allows you to change the CPU quota setting resources.cpu.hard_limit through the API. Elastic Cloud Enterprise uses hard CPU quotas by default, as it is set to true out of the box. If your installation has a high core-to-RAM ratio, you can set this to false to get a better distribution of CPU cores across your deployments.

For more information about the features in Elasticsearch and Kibana 5.6.11 and 6.4.0, check Elasticsearch 5.6.11 Released or Elasticsearch 6.4.0 Released.

Release date: August 23, 2018

Bug fixesedit

ECE version 1.1.5 includes the following bug fixes:

  • Resolved an intermittent issue where Kibana couldn’t connect to the Elasticsearch cluster as it failed to communicate to the proxy component over port 9243.
  • ZooKeeper events are no longer indexed to the Admin Console Elasticsearch cluster. Previously, this could overwhelm the Admin Console Elasticsearch cluster, leading to stability issues.
