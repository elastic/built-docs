Elastic Cloud Enterprise 3.6.2

edit

The following changes are included in this release.

Breaking changes

edit

Integrations Server / APM unable to boot in specific ECE environments. A known issue in the Elastic Stack 8.11.0 release can prevent the Elastic Agent or Integrations Server component from booting up within an ECE deployment. The change affects ECE installations that are deployed with a Linux UID other than 1000. ECE users with deployments that include APM or Integrations Server are recommended to wait for the next patch release, which is planned to include a fix for this problem.

Features

edit

TLS now defaults to ECDHE-RSA-AES256-GCM-SHA384 ciphers only. By default, ECE now only uses ECDHE-RSA-AES256-GCM-SHA384 ciphers to encrypt Transport Layer Security (TLS) communications between ECE components.
