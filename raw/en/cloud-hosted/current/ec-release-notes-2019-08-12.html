Cross-cluster search, OIDC and Kerberos authentication support released

edit

Elastic Cloud now supports additional features that let you do more with our hosted offering:

  • Cross-cluster search template available. The new cross-cluster search (CCS) deployment acts as a hub to manage all of the remote deployment connections. After connecting deployments, you can run queries across multiple Elasticsearch clusters.

    Cross-cluster search is not supported for AWS Marketplace accounts.

  • OpenID Connect support for Elasticsearch and Kibana. To secure your Elasticsearch clusters and Kibana instances, you can now use a OpenID Connect (OIDC) single sign-on authentication.
  • Kerberos support for Elasticsearch and Kibana. For version 7.3 and later, you can use Kerberos-5 authentication protocol to secure your Elasticsearch clusters and Kibana instances.

Bug fixes

edit

We fixed a bug so that manually-added dedicated master nodes now persist through configuration changes.

Master nodes are required only for deployments above a certain size, but you could always add them manually. Before this bug fix, if you created a deployment with dedicated master nodes that didn’t require them, making configuration changes later on could result in those master nodes getting dropped.

Service release: August 12, 2019 (Milestone 25)
