Support to fine-tune snapshot retention periods, open support cases from the Elastic Cloud console, and more features released

edit

Elastic Cloud now supports additional features that let you do more with our hosted offering:

  • Fine-tune your snapshot retention period. In addition to the snapshot interval, you can now also limit the snapshot count, giving you better control over the storage space in your repository. These two settings combine to customize your snapshot retention period.
  • Open support cases from the Elastic Cloud Console. For Trial and Standard subscriptions, you can open a support case from the Elastic Cloud Console by completing the short form. Add a few details and help is on the way!

  • Specify the collection interval and minimum times for monitoring. Several new user settings are supported in this Elastic Cloud release:

    xpack.monitoring.collection.min_interval_seconds
    Indicates the minimum time interval that a chart can represent. The default is 10 seconds.
    xpack.monitoring.kibana.collection.interval
    Controls how often data samples are collected.
    xpack.monitoring.min_interval_seconds
    Indicates the minimum time interval that a chart can represent. Must be the same as the collection interval setting.

Service release: December 17, 2018 (Milestone 14)
