Increased trial capacity, increased scaling limit, and more features released

edit

Elastic Cloud now supports additional features that let you do more with our hosted offering:

  • Review your pricing updates. To give you more control over your usage and spending, we’re providing a more granular billing breakdown for snapshot storage and data transfer costs. These billing changes were released last August and the charges were fully discounted during the promotional period, which ends on February 28, 2019. Learn more …​
  • Bigger trial capacity and increased defaults. Our Elastic Cloud trial now starts with 8 GB of memory for the Elasticsearch cluster, as well as increasing the default settings for templates. All the more room to play with.
  • Increased scaling limit for nodes. For better scaling, we’ve increased the maximum number of nodes possible. Previously at five nodes, the limit has been increased to 10 nodes, per instance configuration. This allows for instance sizes of 10 nodes per availability zone, and in the case of hot/warm architecture, 20 nodes per availability zone.
  • Is it us? We’re asking for feedback if you delete all of your deployments. No problem if you’re just spinning something up later, but if there is something we can improve, let us know!

Service release: February 13, 2019 (Milestone 17)
