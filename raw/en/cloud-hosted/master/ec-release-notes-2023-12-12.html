Enhancements and bug fixes

edit

The following changes are included in this release.

Features

edit

New snapshot dependency API. Added a new endpoint to show an Elasticsearch cluster’s snapshot dependencies.

Improved order of operations for plan changes. Rolling plan changes now happen in the order of data tier: from frozen to hot, followed by master nodes.

Enhancements

edit

Prevent managed keystore override for cloned snapshot repositories. Overriding Elastic-managed keystore entries for cloned repositories could cause issues and is no longer possible.

Bug fixes

edit

Deployment listing with scoped API keys. Fixed an issue where an API key scoped to one or more deployments would get an empty response when using the list deployments API.

Fixed deployment health page. Fixed a bug that would impact the deployment Health page when an Elasticsearch health indicator has an unknown status.
