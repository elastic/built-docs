Node configurationedit

Any setting defined in the elasticsearch.yml configuration file can also be defined for a set of Elasticsearch nodes in the spec.nodeSets[?].config section.

spec:
  nodeSets:
  - name: masters
    count: 3
    config:
      # On Elasticsearch versions before 7.9.0, replace the node.roles configuration with the following:
      # node.master: true
      node.roles: ["master"]
      xpack.ml.enabled: true
      node.remote_cluster_client: false
  - name: data
    count: 10
    config:
      # On Elasticsearch versions before 7.9.0, replace the node.roles configuration with the following:
      # node.master: false
      # node.data: true
      # node.ingest: true
      # node.ml: true
      # node.transform: true
      node.roles: ["data", "ingest", "ml", "transform"]
      node.remote_cluster_client: false

For more information on Elasticsearch settings, see Configuring Elasticsearch.

Marking a node as coordinating-only in ECK requires a YAML type tag due to an issue in a supporting library.

spec:
  nodeSets:
  - name: coordinator
    count: 3
    config:
      node.roles: !!seq ""
