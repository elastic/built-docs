Elastic Cloud on Kubernetes version 2.16.0

edit

Breaking changes

edit
  • The initContainer for Kibana was renamed from elastic-internal-init-config to elastic-internal-init which could cause issues when a user has overridden the podtemplate.spec.initContainers with custom values. The recommendation is to exclude the Kibana resource from reconciliation temporarily during upgrade of ECK and adjust the Kibana manifest to match the new initContainer name. After the ECK upgrade is complete remove the managed annotation from the Kibana manifest. #8426
