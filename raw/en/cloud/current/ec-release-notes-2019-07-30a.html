GCP region Asia Pacific Northeast 1 (Tokyo) availableedit

Elasticsearch Service is now available in the GCP region AP (Tokyo), also known as asia-northeast1. You can select the new region when you create new deployments.

Like other recently added regions, GCP (Tokyo) supports only HTTPS access for additional security. To learn more about what access different regions support, check Elasticsearch Service regions.

Service release: July 30, 2019 (Milestone 24)
