Enhancements and bug fixes - Early October 2024

edit

The following changes are included in this release.

Features

edit

Multifactor authentication (MFA) enabled by default. Elastic Cloud multifactor authentication is now enabled by default for all users. If you don’t have a registered MFA method yet, you will be asked to set one up the next time you log in. New users will automatically get email MFA as part of the signup and email verification process.

Enhancements

edit

Add deployment ID to node certificates. New node certificates now include a SAN with the corresponding deployment ID. This change allows a node to securely present itself as part of the deployment to other services.

Update Beats version to 8.15.2. Upgrade Metricbeat and Filebeat for allocator-metricbeat and beats-runner to version 8.15.2.

Kibana on port 443. All new deployments can use Kibana on port 443 by default, including UI and API. All existing deployments will also have the option to migrate to port 443 through the Cloud UI.

Bug fixes

edit

[Firehose] Group metrics after looping through all records. Metrics are now grouped per request instead of per record. This allows for the handling of multiple records.

[Elastic Cloud MFA] Do not enforce MFA for non-password logins. The MFA setup screen now correctly only shows to users authenticating with username and password, and not with Google/Microsoft or SAML SSO.
