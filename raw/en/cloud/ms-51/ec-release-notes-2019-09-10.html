GCP region Asia Pacific Southeast 1 (Sydney) availableedit

Elasticsearch Service is now available in the GCP region AP (Sydney), also known as australia-southeast1. You can select the new region when you create new deployments.

Like other recently added regions, GCP (Sydney) supports only HTTPS access for additional security. To learn more about what access different regions support, see Elasticsearch Service regions.

Service release: September 10, 2019
