Elastic Cloud Control (ecctl) v1.12.0edit

Bug fixes | Changelog

Welcome to the v1.12.0 release of Elastic Cloud Control. This version brings new features, some breaking changes, and bug fixes.

Elastic Cloud Control (ecctl) is Elastic’s CLI to manage Elasticsearch Service (ESS) and Elastic Cloud Enterprise (ECE).

Bug fixesedit

  • Deployment create command now adds integrations-server by default: Instead of adding APM, for versions >=8.0.0 the default now is integrations-server. This now behaves the same as if the deployment is created via the UI.
  • Fixed allocator vacate tracking for integrations-server: When vacating an integrations-server instance, the command now properly waits until the plan has finished.
