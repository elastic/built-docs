Upgrade your deployment or cluster

edit

Before you upgrade your deployment or cluster to 9.0.0-beta1, ensure you’re prepared to upgrade.

The procedures you follow to upgrade depend on whether you’ve installed Elastic components using Elastic-managed infrastructure or self-managed infrastructure.

If you’re running Elastic-managed infrastructure, your options are:

  • Upgrading on Elastic Cloud Hosted (ECH)
  • Upgrading on Elastic Cloud Serverless (updates are automatic and require no user management)

If you’re running your own self-managed infrastructure—either on-prem or on public cloud—your options are:

  • Upgrading the Elastic Stack (upgrade each component individually)
  • Upgrading on Elastic Cloud Enterprise (ECE)
  • Upgrading on Elastic Cloud on Kubernetes (ECK)

Before you begin upgrading on ECE or ECK, ensure your orchestrator is compatible with 9.0.0-beta1.
