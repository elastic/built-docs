Reimplementing and extending the analyzersedit

The smartcn analyzer could be reimplemented as a custom analyzer that can then be extended and configured as follows:

PUT smartcn_example
{
  "settings": {
    "analysis": {
      "analyzer": {
        "rebuilt_smartcn": {
          "tokenizer":  "smartcn_tokenizer",
          "filter": [
            "porter_stem",
            "smartcn_stop"
          ]
        }
      }
    }
  }
}
