Delete application privileges API

edit

Removes application privileges.

Request

edit

DELETE /_xpack/security/privilege/<application>/<privilege>

Path Parameters

edit
application (required)
(string) The name of the application. Application privileges are always associated with exactly one application.
privilege (required)
(string) The name of the privilege.

Authorization

edit

To use this API, you must have either:

  • the manage_security cluster privilege (or a greater privilege such as all); or
  • the "Manage Application Privileges" global privilege for the application being referenced in the request

Examples

edit

The following example deletes the read application privilege from the myapp application:

DELETE /_xpack/security/privilege/myapp/read

If the role is successfully deleted, the request returns {"found": true}. Otherwise, found is set to false.

{
  "myapp": {
    "read": {
      "found" : true
    }
  }
}
