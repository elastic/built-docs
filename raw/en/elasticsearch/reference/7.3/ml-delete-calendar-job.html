Delete anomaly detection jobs from calendar API

edit

Deletes anomaly detection jobs from a calendar.

Request

edit

DELETE _ml/calendars/<calendar_id>/jobs/<job_id>

Prerequisites

edit
  • If the Elasticsearch security features are enabled, you must have manage_ml or manage cluster privileges to use this API. See Security privileges.

Path parameters

edit
<calendar_id>
(Required, string) Identifier for the calendar.
<job_id>
(Required, string) An identifier for the anomaly detection jobs. It can be a job identifier, a group name, or a comma-separated list of jobs or groups.

Examples

edit

The following example removes the association between the planned-outages calendar and total-requests job:

DELETE _ml/calendars/planned-outages/jobs/total-requests

When the job is removed from the calendar, you receive the following results:

{
   "calendar_id": "planned-outages",
   "job_ids": []
}
