Elasticsearch version 8.13.1edit

Also see Breaking changes in 8.13.

Known issuesedit

  • Due to a bug in the bundled JDK 22 nodes might crash abruptly under high memory pressure. We recommend downgrading to JDK 21.0.2 asap to mitigate the issue.

Bug fixesedit

Aggregations
  • Add test to exercise reduction of terms aggregation order by key #106799
Downsampling
  • Gate reading of optional string array for bwc #106878
Machine Learning
  • Fix Array out of bounds exception in the XLM Roberta tokenizer #106655
Search
  • Fix concurrency bug in AbstractStringScriptFieldAutomatonQuery #106678 (issue: #105911)
  • Fix the background set of significant terms aggregations in case the data is in different shards than the foreground set #106564
Transform
  • Fail checkpoint on missing clusters #106793 (issues: #104533, #106790)

Enhancementsedit

Transform
  • Raise loglevel of events related to transform lifecycle from DEBUG to INFO #106602
