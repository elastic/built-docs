Migrating to 9.x from Enterprise Search 8.x versions

edit

As we continue to evolve and improve Elasticsearch, it’s essential to stay informed about upcoming changes, migrations, and tools that will help us navigate this transition. Below is a list of key resources and initiatives related to App Search migrations, future enhancements, and decommissioning.

Migrating from App Search

edit

As App Search will not be available in 9.0 and beyond, we’ve put together a blog post of Elasticsearch features that you can use. We also created a comparison of App Search and Elasticsearch features in a single handy table, so you can know what to expect.

Beyond App Search: Migrating to Elasticsearch
edit

In our blog post From App Search to Elasticsearch - Tap into the future of search, we outline the benefits of migrating to Elasticsearch including scalability, flexibility, and improved performance. To successfully migrate, developers should start by testing and validating Elasticsearch with a small dataset or single index, using the Elasticsearch API to simplify data transformation, indexing, and query execution.

App Search Migration Notebook
edit

To help with migrating your data to Elasticsearch, we’ve created an executable App Search migration notebook as a guide. The notebook includes step-by-step instructions, migration scripts, and best practices to ensure a smooth transition.

The notebook includes a description of what takes place under the hood when App Search creates a query to Elasticsearch. You’ll find examples of how to filter data during export, including filtering by a specific field value, date range, or multiple conditions. The notebook also includes an example of how to improve your search results by adding semantic search, a powerful Elasticsearch capability which is unavailable in App Search.

Additionally, to help with migrating any App Search Web Crawlers you have to Open Crawler, we’ve created an executable App Search Crawler to Open Crawler Notebook that can convert your App Search Web Crawler configurations into Open Crawler-compliant YAML files. Learn more about Open Crawler.

Migrating from Workplace Search

edit

For users of Workplace Search, we suggest reading the Evolution of Workplace Search: From private data to Elasticsearch blog post to learn more about the transition from Workplace Search to Elasticsearch.

Using Elasticsearch directly gives you access to all the most up-to-date and advanced capabilities developed by Elastic, including AI-powered search. In addition to enabling more customized querying and filtering capabilities, you’ll be able to directly benefit from Elasticsearch’s scalability and performance.

Migrating from Elastic Web Crawler to Open Crawler

edit

Because Elastic Web Crawler will no longer be available in 9.0 and beyond, we have created an Elastic Crawler to Open Crawler Notebook that can convert your Elastic Web Crawler configurations into Open Crawler-compliant YAML files. Learn more about Open Crawler.

Migrating Elastic Managed Connectors

edit

Elastic managed connectors are not supported in 9.0 and beyond. Before upgrading, you must convert all existing managed connectors to self-managed connectors.

We recommend using the Kibana Upgrade Assistant to automatically identify and convert Elastic managed connectors. This lossless process preserves all configurations, schedules, and sync histories. Find the Upgrade Assistant in the Kibana UI using the global search bar.

You can also manually convert connectors through the Connector UI by clicking "Self-manage this connector" in the connector configuration page.

After conversion, deploy these connectors on your own infrastructure. For detailed instructions, refer to the Self-managed connectors documentation.

Elastic Cloud customers will be unable to upgrade until the Kibana Upgrade Assistant can confirm that all your connectors have been converted to self-managed.

Customers using ECK or other self-managed Elastic stacks are strongly encouraged to resolve all Kibana Upgrade Assistant warnings before upgrading, or risk the stability of their upgraded deployment.
