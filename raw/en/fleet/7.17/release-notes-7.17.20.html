Known issues

edit
Fail to bootstrap Fleet Server with self-signed certificates, impacting Elasticsearch Service and self-managed clusters.

Details An issue has been discovered in Fleet Server that causes it to fail to bootstrap with self-signed certificates. This is due to an update in Beats dependencies where the certificate validation was completely rewritten. The issue also prevents bootstrapping Fleet Server in Elasticsearch Service.

This issue is planned to be fixed in version 7.17.21. We recommend that if you’re using Fleet Server you wait until the next version to upgrade.
