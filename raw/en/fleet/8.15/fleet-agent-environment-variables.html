Set environment variables in an Elastic Agent policy

edit

As an advanced use case, you may wish to configure environment variables in your Elastic Agent policy. This is useful, for example, if there are configuration details about the system on which Elastic Agent is running that you may not know in advance. As a solution, you may want to configure environment variables to be interpreted by Elastic Agent at runtime, using information from the running environment.

For Fleet-managed Elastic Agents, you can configure environment variables using the Env Provider. Refer to Variables and conditions in input configurations in the standalone Elastic Agent documentation for more detail.
