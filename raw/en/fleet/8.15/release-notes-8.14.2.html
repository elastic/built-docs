Fleet and Elastic Agent 8.14.2edit

Review important information about the Fleet and Elastic Agent 8.14.2 release.

Notable changesedit

The following are notable, non-breaking updates to be aware of:

Fleet
  • The health_check API endpoint has been updated to accept host IDs. An id string should be used in place of the host field, which is now deprecated. (#185014)

New featuresedit

The 8.14.2 release added the following new and notable features.

Elastic Agent

  • The following processors are now available to users running Elastic Agent in otel mode:

    • Resource Detection Processor #4811
    • Kubernetes Attributes Processor #4893
    • Elastic Infra Metrics Processor #4968
  • An otelcol shortcut has been added for the elasticc-agent otel command. #4816 #4661
  • An agent.monitoring.metrics_period setting has been added, enabling you to control the sampling period of Elastic Agent monitoring metrics according to your needs. #4961

Enhancementsedit

Elastic Agent
  • Add kustomize templates using default manifests for Kubernetes onboarding. #4754 #4657
  • Capture early errors on Windows in the Application eventlog. #4846 #4627

Bug fixesedit

Elastic Agent
  • Fix an issue where the log_writer.go can panic in the case it logs an empty line. #4910 #4907
  • Increase the removal timeout period to 60 seconds when uninstalling Elastic Agent. #4921 #4164
