Fleet and Elastic Agent 8.14.1

edit

Fleet and Elastic Agent 8.14.1edit

Review important information about the Fleet and Elastic Agent 8.14.1 release.

Security updatesedit

Fleet Server
  • Update Fleet Server Go version to 1.21.11. #3607

New featuresedit

The 8.14.1 release added the following new and notable features.

Elastic Agent
  • For users of Elastic Agent running as an OpenTelemetry Collector, the elasticsearchexporter and the filterprocessor are now available to configure in agent pipelines. #4707 #4708

Enhancementsedit

Elastic Agent
  • The more reliable snapshot API is now used in place of the artifact API for snapshot downloads, for both Elastic Agent upgrades to a snapshot and for artifact fetching used in testing. #4693 #4458
  • The Elastic Agent diagnostics bundle now contains an agent-info.yaml file, which provides information on the running agent, including its agent ID, whether or not it’s a snapshot, any headers, and if it’s running in unprivileged mode. #4725 #4439

Bug fixesedit

Fleet
  • The "restart upgrade" button for single agent upgrades is now enabled as expected when the upgrade has been pending for over 2 hours. #184586
Elastic Agent
  • Make Elastic Agent delayed enrollment try indefinitely until the agent is able to enroll successfully into Fleet. #4727 #4716
  • Fix delayed enrollment to work in unprivileged mode on Windows. #4779 #4678
