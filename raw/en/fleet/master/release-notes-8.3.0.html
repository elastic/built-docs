Fleet and Elastic Agent 8.3.0edit

Review important information about the Fleet and Elastic Agent 8.3.0 release.

New featuresedit

The 8.3.0 release adds the following new and notable features.

Fleet
  • Changes to agent upgrade modal to allow for rolling upgrades #132421
Elastic Agent
  • Add ability to set tags during Elastic Agent installation and enrollment. #149 #336
  • Add support for Cloudbeat. #179
  • Add support for Kubernetes cronjobs. #279
  • Increase the download artifact timeout to 10 minutes and add log download statistics. #308
  • Save the agent configuration and the state encrypted on the disk. #535 #398
  • Support scheduled actions and cancellation of pending actions. #393 #419

Enhancementsedit

Fleet
  • Move integration labels below title and normalise styling #134360
  • Adds First Integration Multi Page Steps Flow MVP (cloud only) #132809
  • Optimize package installation performance, phase 2 #131627
  • Adds APM instrumentation for package install process #131223
  • Adds "Label" column + filter to Agent list table #131070
  • Adds cache-control headers to key /epm endpoints in Fleet API #130921
  • Optimize package installation performance, phase 1 #130906
  • Adds experimental features (feature flags) config to fleet plugin #130253
  • Adds redesigned Fleet Server flyout #127786
Elastic Agent
  • Bump node.js version for Heartbeat/synthetics to 16.15.0. #446
  • Add extra k8s resources in clusterRole to better filter objects in dashboards and visualizations. #424
  • Collect Endpoint Security logs on the elastic-agent diagnostics collect command. #105 #242

Fleet
  • Bulk reassign kuery optimize #134673
  • Fixes flickering tabs layout in add agent flyout #133769
  • Adds $ProgressPreference to windows install command in flyout #133756
  • Fixes sorting by size on data streams table #132833
Elastic Agent
  • Elastic Agent now logs stdout and stderr of applications run as processes #88
