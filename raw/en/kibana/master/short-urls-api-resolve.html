Resolve short URL APIedit

[preview] This functionality is in technical preview and may be changed or removed in a future release. Elastic will work to fix any issues, but features in technical preview are not subject to the support SLA of official GA features. Resolve a single Kibana short URL by its slug.

Requestedit

GET <kibana host>:<port>/api/short_url/_slug/<slug>

Path parametersedit

slug
(Required, string) The slug of the short URL.

Response codeedit

200
Indicates a successful call.

Exampleedit

Retrieve the short URL with the hello-world ID:

$ curl -X GET api/short_url/_slug/hello-world

The API returns the following:

{
  "id": "12345",
  "slug": "hello-world",
  "locator": {
    "id": "LOCATOR_ID",
    "version": "x.x.x",
    "state": {}
  },
  "accessCount": 0,
  "accessDate": 1632680100000,
  "createDate": 1632680100000
}
