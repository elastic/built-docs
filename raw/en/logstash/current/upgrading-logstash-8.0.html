Upgrading Logstash to 8.0

edit

Upgrading Logstash to 8.0edit

Before upgrading Logstash:

  • Read the Release Notes.

  • Read the breaking changes docs.

    There you can find info on these topics and more:

    • Elastic Common Schema (ECS) compatibility on by default
    • Secure communication with Elasticsearch
    • Field parser is more strict

If you are installing Logstash with other components in the Elastic Stack, also see the Elastic Stack installation and upgrade documentation.

Upgrading between non-consecutive major versions (6.x to 8.x, for example) is not supported. We recommend that you upgrade to 7.17, and then upgrade to 8.0.

Upgrade to Logstash 7.17 before upgrading to 8.0edit

If you haven’t already, upgrade to version 7.17 before you upgrade to 8.0. If you’re using other products in the Elastic Stack, upgrade Logstash as part of the Elastic Stack upgrade process.

Upgrading to Logstash 7.17 gives you a head-start on new 8.0 features. This step helps reduce risk and makes roll backs easier if you hit a snag.
