Create a monitor status alertedit

Within the Uptime app, create a Monitor Status alert to receive notifications based on errors and outages.

  1. To access this page, go to Observability > Uptime.
  2. On the Overview page, click Alerts > Create alert.
  3. Select Monitor status alert.

If you already have a query in the overview page search bar, it’s populated here.

Conditionsedit

You can specify the following thresholds for your alert.

Status check

Receive alerts when a monitor goes down a specified number of times within a time range (seconds, minutes, hours, or days).

Availability

Receive alerts when a monitor goes below a specified availability threshold within a time range (days, weeks, months, or years).

Let’s create an alert for any monitor that shows Down more than three times in 10 minutes. We’ll choose to only be notified every 30 minutes so that we’re not flooded with alert notifications.

This alert covers all the monitors you have running. You can use a query to specify specific monitors, and you can also have different conditions for each one.

The final step when creating an alert is to select one or more actions to take when the alert is triggered.

Action typesedit

You can extend your alerts by connecting them to actions that use the following supported built-in integrations. Actions are Kibana services or integrations with third-party systems that run as background tasks on the Kibana server when alert conditions are met.

You can configure action types on the Settings page.

Action variablesedit

To customize the notification message, select from a list of variables you would like to include.
