APM version 8.12.0edit

View commits

Breaking Changesedit
  • The unsupported apm_data_stream_migration ingest pipeline has been removed 12102.
  • "publish_ready" is always false in the "GET /" response until events are received by apm-server 12150
  • Reverts a breaking change that ignored the beat name. This is visible in Stack Monitoring 12176
  • A bug has been introduced, so that traces-apm@custom ingest pipeline is now additionally applied to data streams traces-apm.sampled-* and traces-apm.rum-* and applied twice for traces-apm-*. See more details and information around a fix in the Known Issues section.
Bug fixesedit
  • Switch built-in ILM rollover policies from max_size to max_primary_shard_size 11729.
  • Handle empty request bodies gracefully 12242.
  • Wrap processor errors to provide more details 12242.
  • Require only one of exception.{type,message} for log exceptions 12242.
Addededit
  • Update container base image to Ubuntu 22.04 11778
  • Add support for returning partial success response in OTLP input 11883
  • Setting event timestamp from OTel observed timestamp when needed 11935
  • Field mappings have been added for various formerly unindexed fields 12102
  • We now assert that index templates are installed by attempting to create data streams 12150
