WebServer Access Logs Deletededit

Identifies the deletion of WebServer access logs. This may indicate an attempt to evade detection or destroy forensic evidence on a system.

Rule type: eql

Rule indices:

  • auditbeat-*
  • winlogbeat-*
  • logs-endpoint.events.*

Severity: medium

Risk score: 47

Runs every: 5 minutes

Searches indices from: now-9m (Date Math format, see also Additional look-back time)

Maximum alerts per execution: 100

Tags:

  • Elastic
  • Host
  • Linux
  • Windows
  • macOS
  • Threat Detection
  • Defense Evasion

Version: 1

Added (Elastic Stack release): 7.11.0

Rule authors: Elastic

Rule license: Elastic License

Rule queryedit

file where event.type == "deletion" and file.path :
("C:\\inetpub\\logs\\LogFiles\\*.log",
"/var/log/apache*/access.log",
"/etc/httpd/logs/access_log",
"/var/log/httpd/access_log",
"/var/www/*/logs/access.log")
