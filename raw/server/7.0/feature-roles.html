APM Server features that require authorizationedit

After securing APM Server, make sure your users have the roles (or associated privileges) required to use these APM Server features. Note that some of the roles shown here are built-in, and some are user-defined.

Feature Role

Send data to a secured cluster

apm_writer [1]

Load index templates

apm_writer [1] and kibana_user

Read indices created by APM Server

apm_user

View APM Server dashboards in Kibana

kibana_user

To create the user-defined roles shown here, see Configure authentication credentials and Grant users access to APM Server indices. You may want to define additional roles to provide more restrictive access.

[1] These roles are user-defined.
